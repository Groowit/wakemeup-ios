
# 깨워줘 서비스 명세서

## 1. 문서 개요

- 서비스명: 깨워줘
- 문서 목적: 친구끼리 함께 기상하고 서로 깨워줄 수 있는 그룹형 알람 서비스의 제품/기능 명세 정의
- 문서 버전: v0.1
- 작성일: 2026-04-11

## 2. 서비스 한 줄 설명

`깨워줘`는 친구들이 같은 기상 목표를 설정하고, 미션 인증과 친구 음성 알람을 통해 서로 실제로 일어나도록 돕는 소셜 알람 서비스다.

## 3. 문제 정의

기존 알람 앱은 개인 중심이다. 사용자는 알람을 끄고 다시 잠들기 쉽고, 습관 형성에 실패해도 책임감을 느낄 구조가 약하다.

`깨워줘`는 다음 문제를 해결한다.

- 혼자서는 알람을 끄고 다시 자게 되는 문제
- 같은 목표를 가진 친구들과 함께 일어나고 싶은 니즈
- 단순 알람 해제보다 실제 기상 여부를 검증하고 싶은 니즈
- 평범한 알람음보다 더 강한 자극이 필요한 문제

## 4. 핵심 가치 제안

- 함께 일어나기: 친구와 각자 설정한 시간에 일어나는 공동 목표 제공
- 실제 기상 유도: 미션으로 침대 밖 행동 유도
- 감정적 동기 부여: 친구 목소리를 알람으로 사용해 반응성 강화
- 관계 기반 리텐션: 친구와 상호작용할수록 서비스 재사용 유도

## 5. 주요 사용자

### 5.1 타겟 사용자들

- 스터디, 운동, 출근 준비를 함께 하는 친구 그룹
- 생활 습관 개선이 필요한 2~4인 소규모 모임
- 친구들과 일어나고싶은 초,중,고 청소년 학생들

### 5.2 대표 사용자 시나리오

1. 친구 A가 그룹을 만들고 초대 코드로 친구 B, C, D를 초대한다.
2. 그룹원은 매주 각자의 기상시간을 목표로 설정한다.
3. 그룹방장은 미리 정해진 인증 미션 중 하나를 선택한다.
4. 기상 시간이 되면 각자 앱에서 알림을 받는다.
5. 먼저 일어난 친구 A는 미션을 완료하고 `기상 완료` 상태가 된다.
6. B가 아직 미기상 상태라면, A는 B에게 보낼 음성 메시지를 녹음한다.
7. B의 다음 재알림 시점에는 일반 알람음 대신 A의 음성이 재생된다.
8. B도 미션을 완료하면 그룹 내 기상 현황이 갱신된다.
9. 그 뒤 매일 4번부터 8번까지 반복한다.

## 6. 핵심 개념 정의

- 그룹: 함께 기상 목표를 공유하는 친구 모임
- 초대 코드: 그룹 참여를 위한 일회성 또는 일정 시간 유효 코드
- 기상 일정: 그룹이 각자 지킬 기상 시간과 반복 요일
- 미션: 실제 기상 여부를 확인하기 위한 행동 과제
- 기상 세션: 특정 날짜의 실제 기상 이벤트 단위
- 기상 완료: 사용자가 미션 인증까지 마친 상태
- 미기상: 알람 시간이 지났지만 아직 인증하지 않은 상태
- 친구 음성 알람: 먼저 기상한 친구가 미기상 친구에게 보내는 녹음 메시지

## 7. 서비스 범위

### 7.1 MVP 범위

- 회원가입/로그인
- 그룹 생성할때 그룹의 미션 템플릿 선택 및 초대 코드 참여
- 각자의 반복요일과 기상 시간 설정
- 기상 시간 도래 시 알림 발송
- 기상미션 인증 제출
- 그룹 내 기상 현황 표시
- 기상 완료 사용자의 음성 녹음 전송
- 미기상 사용자에게 친구 음성 재알림 제공

### 7.2 제외 범위

- AI 기반 사진 진위 판별
- 음성 변조/편집 기능
- 공개 커뮤니티 기능
- 랭킹/포인트/배지 고도화
- 웨어러블 디바이스 연동

## 8. 기능 요구사항

### 8.1 회원가입 및 인증

### 목적

사용자를 식별하고 친구 기반 그룹 참여를 가능하게 한다.

### 요구사항

- 사용자는 휴대폰 번호 또는 소셜 로그인으로 가입할 수 있어야 한다.
- 가입 시 닉네임, 프로필 이미지(선택)를 설정할 수 있어야 한다.
- 푸시 알림 권한을 요청해야 한다.
- 마이크 권한은 실제 기능 사용 시점에 요청해야 한다.

### 8.2 그룹 생성 및 초대

### 목적

친구끼리 폐쇄형 그룹을 쉽게 만들고 참여할 수 있도록 한다.

### 요구사항

- 사용자는 그룹을 생성할 수 있어야 한다.
- 그룹 생성자는 그룹명과 기본 기상 시간을 입력할 수 있어야 한다.
- 시스템은 초대 코드와 초대링크를 생성해야 한다.
- 사용자는 초대 코드를 입력하거나 초대링크를 클릭해 참여할 수 있어야 한다.
- 한 사용자는 여러 그룹에 참여하려면 유료버전을 결제해야한다. (MVP 범위를 벗어남→ 제외)
- 그룹당 최대 인원은 MVP 기준 4명으로 제한한다.
- 만약 그룹당 최대인원을 5명이상으로 늘리고싶다면 유료버전을 결제해야한다. (MVP 범위를 벗어남→ 제외)
- 그룹은 기준 타임존을 가져야 하며, 기본값은 생성자의 디바이스 타임존으로 설정한다.
- 초대 코드는 만료 시간 또는 최대 사용 횟수를 가질 수 있어야 한다.
- 그룹 생성자는 그룹 설정 수정 권한을 가진다.

### 8.4 미션 설정

### 목적

사용자가 침대에서 벗어나 실제 행동을 하도록 유도한다.

### 요구사항

- 그룹은 기상 일정마다 1개의 기본 미션을 설정할 수 있어야 한다.
- 그룹기상미션은 최초로 방장이 그룹을 생성할때 선택할 수 있다.(팀원은 선택할 수 없다)
- MVP 미션 예시는 아래와 같다.
    - 타자치기 (방장이 이 미션을 선택하면, 각자 팀원은 타자치기의 문장을 적을 수 있고, 실제 송출되는 문장은 그룹원이 작성한 문장 중 랜덤으로 선택된다. 만약 아무도 문장을 적지 않는다면 랜덤명언 문장이 기본으로 송출된다)
        - 사람들은 알람을 설정하는 시간 때 문장을 필수로 적을 수 있어야 한다.
        - 알람을 세팅할 때 '문장'을 하나 적어두면, 그걸 오타 없이 똑같이 쳐야 알람이 꺼지는 방식
        - **시스템:** 화면에 "나는 오늘 지각하면 [친구 이름]에게 소고기를 사겠습니다" 같은 문장이 뜨고, 밑에 `TextInput`을 띄워. 띄어쓰기 하나라도 틀리면 안 꺼짐.
    - 통통 튀는 버튼 10개 잡기
        - 알람 끄기 버튼이 가만히 있질 않고 화면 여기저기로 이동함.
        - 알람 버튼을 연속적을 성공할 때마다 속도가 점차 빨라짐.
        - **시스템:** '알람 끄기' 버튼을 누르기 위해서는 이동하는 버튼을 눌러야 함.이걸 10번 성공적으로 명중시켜야 알람이 꺼짐
    - 화면 100번 연타하기
    - 흔들기 미션의 완벽한 UI 버전
    - **시스템:** 화면 중앙에 거대한 배터리나 게이지가 있어. 유저가 화면을 미친 듯이 다다다다 터치해서 이 게이지를 100%까지 꽉 채워야만 알람이 꺼져. (단, 1초라도 터치를 쉬면 게이지가 다시 깎임!)
- 미션 제출 시 제출 시각, 그룹, 세션 정보가 함께 저장되어야 한다.
- 인증이 정상 제출되면 사용자 상태는 `기상 완료`로 변경된다.

### 8.6 알림 및 알람 동작

### 목적

정해진 시간에 사용자가 확실히 인지할 수 있도록 알림을 제공한다.

### 요구사항

- 기상 시간 도래 시 사용자에게 푸시 또는 로컬 알림을 발송해야 한다.
- 앱이 열려 있는 경우 인앱 알람 UI를 노출할 수 있어야 한다.
- 앱 내 기상 플로우에서는 사용자가 미션 완료 또는 다시 알림 선택 흐름으로 자연스럽게 이동하도록 설계해야 한다.
- 다시 알림은 5분 간격으로 최대 3회 제공한다.
- 친구 음성 알람이 도착한 경우 다음 재알림부터 우선 재생한다.

### 참고

모바일 OS 정책상 기본 시스템 알람을 완전히 대체하는 동작은 플랫폼별 제약이 있을 수 있다. 따라서 MVP는 `앱 알림 + 반복 재알림 + 친구 음성 재생` 중심으로 설계하고, 실제 배포 전 iOS/Android 정책 검토가 필요하다.

### 8.7 친구 음성 알람

### 목적

이미 일어난 친구가 아직 일어나지 않은 친구를 직접 깨울 수 있게 한다.

### 요구사항

- `기상 완료` 상태의 사용자는 같은 그룹의 `미기상` 사용자에게 음성 메시지를 보낼 수 있어야 한다.
- 음성 메시지는 미기상 사용자 모두에게 보내는 것을 MVP 기본으로 한다.
- 녹음 길이는 최대 15초로 제한한다.
- 음성 메시지는 그날 하루동안의 미기상 팀원에게만 유효하다.
- 기상한 사용자는 미기상 팀원들에게 최대 1개의 음성 메시지를 보낼 수 있다.
- 기상팀원이 여러명이라서 여러 메시지가 존재하면 가장 최근 메시지를 다음 재알림에 우선 사용한다.
- 수신자는 알람 재생 시 발신자 이름을 함께 볼 수 있어야 한다.
- 음성 메시지는 저장 전 악성/공격적 사용에 대비해 신고 기능 확장을 고려해야 한다.

### 8.8 그룹 현황판

### 목적

누가 일어났고 누가 아직 안 일어났는지 그룹이 한눈에 확인할 수 있게 한다.

### 요구사항

- 그룹 홈에서 오늘의 기상 세션 상태를 볼 수 있어야 한다.
- 각 그룹원은 아래 상태 중 하나로 표기되어야 한다.
    - 기상 전
    - 알림 진행 중
    - 기상 완료
    - 실패
- 그룹원은 누가 누구에게 음성 알람을 보냈는지 확인할 수 있어야 한다.
- 기상 완료 시간은 그룹원에게 공유할 수 있어야 한다.

### 8.9 히스토리

### 목적

사용자와 그룹의 습관 형성을 돕는다.

### 요구사항

- 사용자는 최근 7일 또는 30일 기상 성공 기록을 볼 수 있어야 한다.
- 그룹은 공동 성공률을 볼 수 있어야 한다.
- MVP에서는 간단한 달력형 또는 리스트형 히스토리만 제공한다.

## 9. 주요 사용자 플로우

### 9.1 그룹 생성 플로우

1. 사용자가 앱 가입 후 `그룹 만들기` 선택
2. 그룹명, 미션 선택
3. 서버가 그룹과 초대 코드 생성
4. 생성자는 초대 코드를 친구에게 공유
5. 친구는 코드 입력 후 그룹 참여

### 9.2 기상 인증 플로우

1. 기상 시간 도래
2. 사용자에게 알림 발송
3. 사용자가 앱 진입
4. 미션 확인
5. 미션 수행
6. 미션 인증
7. 서버 검증 후 `기상 완료` 처리
8. 그룹 현황판 즉시 갱신

### 9.3 친구 음성 알람 플로우

1. 친구 A가 먼저 기상 완료
2. 그룹 현황판에서 미기상 친구 B 확인
3. A가 B에게 음성 녹음
4. 서버에 음성 파일 업로드 및 세션 연결
5. B의 다음 재알림 시 해당 음성 재생
6. B가 기상 완료하면 더 이상 재생하지 않음
