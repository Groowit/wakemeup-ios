# 깨워줘 화면 흐름 및 UX 설계서

## 1. 문서 목적

- 이 문서는 `PRODUCT.md`를 바탕으로 실제 앱 화면이 어떤 순서로 연결되고, 각 화면이 어떤 역할을 해야 하는지 정리한 UX 설계 문서다.
- `PRODUCT.md`가 기능과 정책 중심 문서라면, 이 문서는 사용자가 실제로 보게 되는 화면 흐름과 인터랙션 중심 문서다.
- 디자인 시안과 구현 화면 사이의 공통 기준점을 만드는 것을 목표로 한다.

## 2. 문서 활용 방식

- 기능 요구사항의 기준 문서는 `PRODUCT.md`를 따른다.
- 화면 구성, 진입 조건, CTA 우선순위, 상태값 정의는 이 문서를 따른다.
- 시안 변경이나 구현 변경이 생기면 `PRODUCT.md`와 함께 이 문서도 같이 업데이트한다.

## 4. 전체 사용자 흐름

### 4.1 최초 진입 흐름

`온보딩 → 회원가입/프로필 설정 → 그룹 없음 홈 → 그룹 만들기 또는 초대 코드 참여`

### 4.2 그룹 생성 흐름

`그룹 없음 홈 → 그룹 생성 → 미션 선택 → 초대 코드 공유 → 내 기상 시간 설정 → 그룹 홈`

### 4.3 그룹 참여 흐름

`그룹 없음 홈 또는 초대 링크 진입 → 그룹 참여 → 내 기상 시간 설정 → 그룹 홈`

### 4.4 일일 기상 흐름

`알림 수신 → 알람 진입 화면 → 친구 음성 알람 수신(있는 경우) → 미션 수행 → 기상 완료 → 친구 깨우기 또는 그룹 현황 확인`

## 5. 공통 UX 원칙

- 모든 화면에는 사용자가 바로 이해할 수 있는 메인 CTA 1개를 둔다.
- 홈 화면에서는 "오늘 내가 무엇을 해야 하는지"가 가장 먼저 보여야 한다.
- 알람, 미션, 음성 재생 화면은 몰입을 방해하는 요소를 최대한 줄인다.
- 푸시 권한은 기상 시간 저장 직전에 요청한다.
- 마이크 권한은 음성 녹음 시작 직전에 요청한다.
- 설명은 짧게, 상태와 시간과 이름은 크게 보여준다.
- 같은 정보를 여러 화면에 반복 배치하지 않는다.
- 탭 화면과 풀스크린 플로우 화면을 역할별로 분리한다.

## 7. 화면별 상세 설계

### 7.1 온보딩

#### 목적

- 서비스의 차별점인 "친구와 함께 깨는 소셜 알람"을 짧고 명확하게 전달한다.

#### 핵심 질문

- 이 앱이 일반 알람 앱과 무엇이 다른가?
- 왜 친구와 함께 일어나는 것이 중요한가?

#### 구성 요소

- 로고
- 대표 이미지 또는 일러스트
- 핵심 메시지 1줄
- 보조 설명 2줄
- 페이지 인디케이터
- `시작하기` 버튼
- `이미 계정이 있나요? 로그인` 텍스트 버튼

#### UX 포인트

- 온보딩은 3장 이내로 끝낸다.
- 기능 소개보다 가치 제안을 먼저 보여준다.
- 권한 요청은 이 화면에서 하지 않는다.

### 7.2 회원가입 / 프로필 설정

#### 목적

- 최소 정보만 받고 빠르게 앱 핵심 흐름으로 진입시킨다.

#### 구성 요소

- 프로필 이미지 선택
- 닉네임 입력
- 소셜 로그인 또는 간편 시작 버튼

#### 필수/선택 정보

- 필수: 닉네임
- 선택: 프로필 이미지

#### UX 포인트

- 처음부터 많은 정보를 요구하지 않는다.
- 프로필 이미지는 건너뛸 수 있게 한다.
- 친구에게 보여질 화면이라는 맥락을 짧게 설명한다.

### 7.3 그룹 없음 홈

#### 목적

- 사용자가 바로 다음 행동을 선택하게 만드는 화면이다.

#### 메인 액션

- `그룹 만들기`
- `초대 코드로 참여하기`

#### 구성 요소

- 빈 상태 메시지
- 생성/참여 액션 카드 2개
- 서비스 사용 방식에 대한 짧은 안내

#### UX 포인트

- 버튼보다 큰 카드형 액션이 더 직관적이다.
- 초대 링크로 진입한 사용자는 이 화면을 거치지 않고 참여 화면으로 바로 연결하는 것이 좋다.

### 7.4 그룹 생성

#### 목적

- 새 그룹의 기본 정체성을 정한다.

#### 구성 요소

- 단계 표시 `Step 1/3`
- 그룹 이름 입력
- 그룹 설명 문구
- 예상 멤버 수 선택
- `다음` 버튼

#### UX 포인트

- 설정은 단순해야 한다.
- 타임존 같은 고급 설정은 기본값 자동 적용 후 숨긴다.
- 입력 항목은 최소화한다.

### 7.5 미션 선택

#### 목적

- 그룹의 기상 인증 방식을 방장이 정하게 한다.

#### 구성 요소

- 단계 표시 `Step 2/3`
- 미션 카드 리스트
- 미션별 아이콘
- 미션 한 줄 설명
- 선택 상태 표시
- `선택 완료` 버튼

#### UX 포인트

- 한 번에 하나만 선택 가능하게 한다.
- 미션마다 "왜 잠이 깨는지"가 감각적으로 이해돼야 한다.
- 타자치기 미션은 이후 개인 문장 입력이 필요하다는 안내가 있어야 한다.

### 7.6 초대 코드 공유

#### 목적

- 그룹 생성 직후 친구 초대를 자연스럽게 유도한다.

#### 구성 요소

- 생성 완료 메시지
- 초대 코드 카드
- `초대 코드 복사`
- `친구에게 초대 링크 보내기`
- `알람 시간 설정하러 가기`

#### UX 포인트

- 공유를 유도하되 강제하지 않는다.
- 초대가 아직 안 되어도 다음 단계로 진행 가능해야 한다.
- 이 화면은 성취감과 다음 행동 제시를 동시에 해야 한다.

### 7.7 그룹 참여

#### 목적

- 받은 초대 코드로 빠르고 쉽게 그룹에 합류하게 한다.

#### 구성 요소

- 6자리 코드 입력 박스
- 붙여넣기 지원
- `참여하기`
- `초대 코드가 무엇인가요?`

#### UX 포인트

- 입력 시 자동으로 다음 칸으로 이동해야 한다.
- 오류는 토스트보다 인라인 메시지로 보여주는 편이 낫다.
- 코드가 만료된 경우 대체 행동을 안내해야 한다.

### 7.8 내 기상 시간 설정

#### 목적

- 사용자별 알람 시간과 반복 요일을 정하게 한다.

#### 구성 요소

- 알람 on/off
- 시간 선택
- 반복 요일 선택
- 알람음 선택
- 진동 선택
- `설정 완료하기`

#### UX 포인트

- 푸시 권한은 이 화면에서 저장 직전에 요청한다.
- 타자치기 미션인 경우 개인 문장 입력 영역을 이 화면 하단에 함께 두는 것이 자연스럽다.
- 저장 후에는 바로 그룹 홈으로 이동시킨다.

### 7.9 그룹 홈

#### 목적

- 오늘의 기상 상태와 현재 가능한 행동을 한 번에 보여주는 핵심 화면이다.

#### 가장 먼저 보여야 하는 정보

- 현재 시각
- 내 알람/내 상태
- 그룹 전체 진행 상황
- 멤버별 상태

#### 구성 요소

- 현재 시각 카드
- 그룹 성공률 카드
- 오늘의 미션 정보
- 그룹 멤버 리스트
- 초대 코드 또는 그룹 관리 진입점

#### 상태별 메인 CTA

- 알람 전: `내 알람 수정`
- 알림 진행 중: `미션 시작하기`
- 기상 완료: `친구 깨우기`
- 모두 완료: `오늘 기록 보기`

#### UX 포인트

- 홈의 핵심은 정보 나열이 아니라 행동 유도다.
- 현재 시안의 `+` 버튼은 의미가 모호하므로 명시적 액션 버튼으로 바꾸는 것이 좋다.
- 멤버 상태는 색과 라벨만으로 빠르게 구분되어야 한다.

### 7.10 알람 진입 화면

#### 목적

- 사용자를 잠에서 깨어난 행동 상태로 빠르게 전환시킨다.

#### 구성 요소

- 현재 시간
- 알람 상태 메시지
- 큰 아이콘 또는 시각적 자극 요소
- `미션 시작하기`
- `5분 뒤 다시 알림`

#### UX 포인트

- 탭바와 기타 설정 요소는 숨긴다.
- 이 화면은 판단보다 행동을 유도해야 한다.
- 정보는 최소화하고 버튼은 크게 둔다.

### 7.11 친구 음성 알람 수신

#### 목적

- 친구의 목소리로 감정적 자극과 긴장감을 준다.

#### 구성 요소

- 발신자 프로필 이미지
- 발신자 이름
- 음성 재생 상태
- `기상미션 하러가기`
- `5분만 더 자기`

#### UX 포인트

- 음성은 자동 재생 후 재생 상태를 명확히 보여준다.
- 여러 음성이 있을 경우 최신 음성을 우선 재생한다.
- 감정적 몰입은 높이되 조작은 단순해야 한다.

### 7.12 미션 수행

#### 목적

- 실제로 손과 몸을 움직여 잠에서 깨게 만든다.

#### 공통 구성 요소

- 현재 단계 또는 진행률
- 미션 이름
- 큰 인터랙션 영역
- 즉각적인 피드백

#### 미션별 요구사항

##### 타자치기

- 목표 문장 카드
- 입력창
- 글자 수 또는 진행 표시
- 오타 즉시 피드백

##### 100번 연타

- 중앙 게이지
- 남은 횟수 표시
- 멈추면 게이지가 감소하는 피드백

##### 도망가는 버튼 잡기

- 전체 화면 플레이 영역
- 현재 성공 횟수 표시
- 버튼 이동 속도 조절

#### UX 포인트

- 미션은 재미보다 "잠이 깨는 감각"이 우선이어야 한다.
- 실패 피드백은 자극적이되 불쾌하지 않게 만든다.
- 잘못 눌러도 다시 시도하기 쉬워야 한다.

### 7.13 기상 완료

#### 목적

- 사용자가 미션 완료 성취감을 느끼고 다음 행동으로 이동하게 한다.

#### 구성 요소

- 축하 비주얼
- 기상 완료 시각
- 메인 CTA
- 보조 CTA

#### 권장 CTA

- 아직 안 일어난 친구가 있으면 `친구 깨우기`
- 모두 완료되었으면 `그룹 현황 확인하기`
- 보조 버튼은 `나중에 확인하기`

#### UX 포인트

- 이 화면은 짧고 강하게 끝내는 것이 좋다.
- 완료 보상은 분명해야 하지만 오래 머무는 화면일 필요는 없다.

### 7.14 친구 깨우기 음성 녹음

#### 목적

- 먼저 일어난 친구가 미기상 친구를 바로 깨울 수 있게 한다.

#### 구성 요소

- 받는 사람 목록
- 녹음 시간 표시 `00:00 / 00:15`
- 파형 시각화
- 녹음 버튼
- `친구에게 전송하기`
- `다시 녹음하기`

#### UX 포인트

- 마이크 권한은 이 화면 진입 직후가 아니라 녹음 버튼을 누르는 순간 요청한다.
- 1초 미만 녹음은 전송 불가 처리하는 것이 좋다.
- 수신 대상은 자동 선택하되 누가 받는지 명확히 보여줘야 한다.

### 7.15 기록

#### 목적

- 기상 습관이 쌓이고 있다는 보상을 시각적으로 제공한다.

#### 구성 요소

- 최근 7일 성공률
- 그룹 성공률
- 월간 캘린더
- 인사이트 카드

#### UX 포인트

- 사용자는 수치보다 변화 추세에 더 반응한다.
- "지난주보다 좋아졌어요" 같은 해석형 메시지가 중요하다.
- 실패 기록도 숨기지 말고 부담 없이 보이게 해야 한다.

### 7.16 프로필

#### 목적

- 사용자 정보와 설정 진입점을 관리한다.

#### 구성 요소

- 프로필 이미지
- 닉네임
- 이번 주 성공률
- 연속 기상 일수
- 설정 메뉴 리스트

#### 권장 메뉴

- 프로필 수정
- 알림/진동 설정
- 그룹 설정
- 로그아웃

#### UX 포인트

- 관리 기능은 홈보다 프로필 탭에 모으는 편이 명확하다.
- 홈 화면은 오늘 행동 중심, 프로필은 설정 중심 역할을 갖는다.

## 8. 화면 우선순위

- 친구 깨우기 음성 녹음 화면

### 8.2 그 다음으로 중요한 화면

- 그룹 생성
- 그룹 참여
- 내 기상 시간 설정
- 기록

## 9. 정리

- 이 서비스의 핵심은 화면 수를 늘리는 것이 아니라, 그룹 홈과 알람 플로우를 강하게 만드는 것이다.
- 따라서 각 화면은 예쁘게 만드는 것보다 "지금 이 사용자에게 무엇을 하게 할 것인가"가 먼저 보이도록 설계해야 한다.
- 이후 구현 단계에서는 이 문서를 기준으로 화면별 컴포넌트와 상태 전환을 세분화한다.
